## 1. 本次更新说明

本次更新将测试用例同步到 PRD、DESIGN.md 与 TECH_SPEC.md 当前结论：HomeCare Agent v0.1 是 **移动端单一产品**，采用 **Agent-first 主界面 + ChatGPT 式左侧抽屉 + 底部 Composer**，不使用传统底部五 Tab 导航。

本次调整包括：

1. 删除旧版「首页｜设备｜Agent｜提醒｜更多」五 Tab 导航相关验收项。
2. 删除「更多页」作为一级页面的旧结构，改为左侧抽屉中的低频管理入口。
3. 将默认首屏统一为「Agent 主界面 / Agent Home」，不再拆成传统首页与 Agent Tab。
4. 新增左侧抽屉打开、关闭、跳转、当前入口高亮、无底部导航可发现性的测试用例。
5. 将设备库、提醒、Agent 执行记录、设置、关于统一改为从左侧抽屉进入；数据导出收纳在设置页，不作为抽屉独立入口。
6. 保留并强化手机拍照 / 相册 / 文件上传、底部弹窗、折叠面板、确认卡片等移动端交互测试。
7. 保留并强化 Agent 写入前确认、安全边界、说明书不编造、高风险维修拒绝指导等关键测试。
8. 新增设备选择态、保存故障记录确认态、上传失败恢复态、无说明书 / 未检索到答案态等遗漏测试。
9. MVP 验收标准统一改为：移动端 Agent-first 闭环 + 左侧抽屉 + 无传统底部导航。

## 2. 测试目标

验证 HomeCare Agent v0.1 是否满足移动端 Agent-first MVP 需求，重点验证：

1. 用户打开应用后首先进入 Agent 主界面，可以直接输入任务、上传资料、查看 Agent 状态和结构化结果。
2. 应用使用顶部栏、左侧抽屉和底部 Composer，不展示传统底部五 Tab 导航。
3. 用户可以通过手机拍照、相册图片或 PDF 文件上传发起建档任务。
4. Agent 可以识别 create_device、manual_qa、warranty_check、troubleshooting 四类意图。
5. Agent 可以解析上传资料，生成设备档案草稿，并在写入前等待用户确认。
6. 用户可以在移动端确认、修改或取消设备草稿。
7. 用户确认后，系统可以创建设备档案、绑定附件、计算保修状态、创建应用内提醒并索引说明书。
8. 用户可以从左侧抽屉进入设备库、提醒、Agent 执行记录、设置等低频页面。
9. 用户可以在设备列表和设备详情页查看设备资料。
10. 用户可以基于说明书向 Agent 提问，且系统不得编造说明书中不存在的内容。
11. 用户可以发起故障售后辅助流程，获得安全提醒、排查步骤、售后沟通文案和材料清单。
12. 保存故障记录、创建设备、绑定附件、创建提醒、删除设备等写入或高风险行为必须经过用户确认。
13. 用户可以查看 Agent 的当前执行过程和历史执行记录。
14. 移动端单栏布局、底部 Composer、底部弹窗、确认卡片、折叠内容符合移动端使用习惯。
15. 高风险维修请求必须被保守处理，不得指导带电维修、燃气维修、复杂拆机维修等危险行为。

## 4. 测试优先级定义

| 优先级 | 含义 | 示例 |
|---|---|---|
| P0 | MVP 必须通过，否则核心闭环不可验收 | 登录鉴权、Agent 主界面、无底部导航、左侧抽屉、自动建档、草稿确认、确认后入库、说明书不编造、故障售后安全边界 |

## 25. MVP 验收场景

### 25.1 场景 A：移动端自动建档闭环

**前置条件：**

1. 手机端可打开 HomeCare Agent。
2. 准备一张订单截图和一份说明书 PDF。

**操作步骤：**

1. 用户进入 Agent 主界面。
2. 点击 Composer 左侧「+」。
3. 选择拍照 / 相册上传订单截图。
4. 选择文件上传说明书 PDF。
5. 输入：

```text
帮我把这个设备建档
```

6. 提交 Agent 任务。
7. 查看执行状态。
8. 查看设备草稿确认卡片。
9. 如有错误，点击「修改信息」在底部弹窗中修正。
10. 点击「确认创建」。
11. 点击「查看设备」。

**预期结果：**

1. 系统识别为 create_device 意图。
2. 文件上传成功并进入解析流程。
3. Agent 提取设备信息。
4. 系统生成设备草稿。
5. 系统暂停等待用户确认。
6. 用户确认后系统创建设备。
7. 系统绑定附件。
8. 系统计算保修截止日期。
9. 系统创建应用内保修提醒。
10. 系统索引说明书。
11. 设备出现在设备库。
12. 设备详情页能查看该设备。
13. 执行记录中能看到完整路径。

---

### 25.2 场景 B：移动端说明书问答闭环

**前置条件：**

1. 设备已建档。
2. 设备已绑定并索引说明书。

**操作步骤：**

1. 从左侧抽屉进入设备库。
2. 进入设备详情页。
3. 点击「问说明书」。
4. 输入：

```text
这个设备滤芯怎么清洗？
```

**预期结果：**

1. 系统自动带入当前设备上下文。
2. 系统识别为 manual_qa。
3. 系统检索该设备说明书。
4. 系统基于说明书生成回答。
5. 系统显示答案来源。
6. 如果没有说明书，必须明确提示未上传说明书，不得编造。
7. 如果说明书中没有答案，必须提示未找到明确答案，不得编造。

---

### 25.3 场景 C：移动端故障售后闭环

**前置条件：**

1. 已有一台净水器设备。
2. 设备有保修信息。
3. 设备已绑定说明书。

**操作步骤：**

1. 从左侧抽屉进入设备库。
2. 进入设备详情页。
3. 点击「故障售后」。
4. 输入：

```text
我的净水器漏水了，帮我看看怎么办。
```

5. 查看 Agent 输出结果。
6. 点击「售后沟通文案」查看文案 Sheet。
7. 返回故障结果，点击保存故障记录。
8. 查看保存确认卡片。
9. 点击确认保存。

**预期结果：**

1. 系统识别为 troubleshooting。
2. 系统找到当前设备。
3. 系统判断保修状态。
4. 系统检索说明书。
5. 系统生成安全提醒。
6. 系统生成排查步骤。
7. 系统生成售后沟通文案。
8. 系统列出材料清单。
9. 系统询问是否保存故障记录。
10. 用户确认后保存故障记录。
11. 故障记录出现在设备详情页维修 / 故障 / 保养记录区。

---

### 25.4 场景 D：移动端 Agent-first 体验验收

**操作步骤：**

1. 使用手机打开应用。
2. 确认默认进入 Agent 主界面。
3. 打开并关闭左侧抽屉。
4. 从抽屉依次进入设备库、提醒、Agent 执行记录、设置。
5. 回到 Agent 主界面完成一次上传资料建档。
6. 查看设备库和设备详情。
7. 发起说明书问答。
8. 发起故障售后。
9. 查看 Agent 执行记录。

**预期结果：**

1. 移动端可以完成上传资料建档。
2. 移动端可以查看设备库。
3. 移动端可以查看设备详情。
4. 移动端可以发起说明书问答。
5. 移动端可以发起故障售后。
6. 移动端左侧抽屉可正常打开、关闭和跳转。
7. 移动端不展示传统底部五 Tab 导航。
8. 移动端确认卡片、底部弹窗、上传组件体验正常。
9. 移动端不出现需要 PC 宽屏才能操作的布局。
10. 移动端页面不依赖复杂表格。
11. 移动端关键任务路径可单手完成。
